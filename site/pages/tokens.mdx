# Tokens

## Overview

The `viem/tokens` entrypoint provides canonical, type-safe abstractions for stablecoins & tokens. 

Extend a Client with a token decorator (e.g. `usdc()`) to attach a token namespace (e.g. `client.usdc`). 

```ts twoslash
import { createClient, http } from 'viem'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({
  chain: mainnet,
  transport: http(),
}).extend(usdc())

const balance = await client.usdc.getBalance({
  account: '0x55FE002aefF02F77364de339a1292923A15844B8',
})
```

Tokens are built on a single composable factory ([`Decorators.defineErc20`](#define-a-custom-token)), so new
tokens are added by supplying an address map, with no duplicated action code.

## Recipes

These recipes assume you have [set up a Client](/docs/clients/intro) with a `chain`, extended with
the `usdc()` decorator.

### Read a Balance

Read an account's token balance with `getBalance`. The result is in the token's base units.

```ts twoslash
import { createClient, http } from 'viem'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({
  chain: mainnet,
  transport: http(),
}).extend(usdc())

const balance = await client.usdc.getBalance({
  account: '0x55FE002aefF02F77364de339a1292923A15844B8',
})
```

### Read Token Metadata

Read `name`, `symbol`, `decimals`, and `totalSupply` in a single multicall with `getMetadata`.

```ts twoslash
import { createClient, http } from 'viem'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({
  chain: mainnet,
  transport: http(),
}).extend(usdc())

const { name, symbol, decimals, totalSupply } = await client.usdc.getMetadata()
```

### Transfer Tokens

Move tokens from your account to another with `transferSync`, which waits for the transaction to be
confirmed and returns the receipt plus the `Transfer` event. Pass `amount` as a human-readable
decimal string, parsed with the token's `decimals`.

```ts twoslash
import { createClient, http } from 'viem'
import { privateKeyToAccount } from 'viem/accounts'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({
  account: privateKeyToAccount('0x...'),
  chain: mainnet,
  transport: http(),
}).extend(usdc())

const { receipt, value } = await client.usdc.transferSync({
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  amount: '10.5', // 10.5 USDC
})
```

Use `transfer` instead of `transferSync` if you only want the transaction hash without waiting for
confirmation.

### Approve a Spender

Authorize another account to spend your tokens with `approveSync`, then read the remaining allowance
with `allowance`.

```ts twoslash
import { createClient, http } from 'viem'
import { privateKeyToAccount } from 'viem/accounts'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const account = privateKeyToAccount('0x...')
const client = createClient({
  account,
  chain: mainnet,
  transport: http(),
}).extend(usdc())

await client.usdc.approveSync({
  spender: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  amount: '10.5',
})

const allowance = await client.usdc.allowance({
  owner: account.address,
  spender: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
})
```

### Transfer From an Approved Account

Move tokens on behalf of an owner that has approved you by passing `from` to `transferSync`
(this calls `transferFrom` under the hood).

```ts twoslash
import { createClient, http } from 'viem'
import { privateKeyToAccount } from 'viem/accounts'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({
  account: privateKeyToAccount('0x...'),
  chain: mainnet,
  transport: http(),
}).extend(usdc())

const { receipt } = await client.usdc.transferSync({
  from: '0x55FE002aefF02F77364de339a1292923A15844B8',
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  amount: '10.5',
})
```

### Estimate Gas for a Write

Every write action exposes a `.call` helper that returns a contract call, ready to pass to actions
like [`estimateContractGas`](/docs/contract/estimateContractGas) or
[`simulateContract`](/docs/contract/simulateContract). The address is resolved from the client
chain, or pass `address` to override it.

```ts twoslash
import { createClient, http } from 'viem'
import { estimateContractGas } from 'viem/actions'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({
  chain: mainnet,
  transport: http(),
}).extend(usdc())

const gas = await estimateContractGas(client, {
  ...client.usdc.transfer.call({
    to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
    amount: '1',
  }),
  account: '0x55FE002aefF02F77364de339a1292923A15844B8',
})
```

### Batch Calls

Use `.call` to compose multiple token operations into a single batched transaction with
[`sendCalls`](/docs/actions/wallet/sendCalls).

```ts twoslash
import { createClient, http, walletActions } from 'viem'
import { privateKeyToAccount } from 'viem/accounts'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({
  account: privateKeyToAccount('0x...'),
  chain: mainnet,
  transport: http(),
})
  .extend(walletActions)
  .extend(usdc())

const { id } = await client.sendCalls({
  calls: [
    client.usdc.approve.call({
      spender: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
      amount: '1',
    }),
    client.usdc.transfer.call({
      to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
      amount: '1',
    }),
  ],
})
```

### Watch Transfers

Subscribe to `Transfer` (or `Approval`) events with `watchTransfer`. It returns a function to
unsubscribe.

```ts twoslash
import { createClient, http } from 'viem'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({
  chain: mainnet,
  transport: http(),
}).extend(usdc())

const unwatch = client.usdc.watchTransfer({
  onTransfer: (args, log) => {
    console.log('Transfer:', args.from, '→', args.to, args.value)
  },
})
```

### Look Up a Token Address

Resolve a token address with `getAddress`. On a bound token it defaults to the client chain; pass a
chain id (or client) to override. It throws if the token is not deployed on the chain.

```ts twoslash
import { createClient, http } from 'viem'
import { mainnet, optimism } from 'viem/chains'
import { Addresses, Decorators, usdc } from 'viem/tokens'

const client = createClient({
  chain: mainnet,
  transport: http(),
}).extend(usdc())

client.usdc.getAddress() // resolve from client chain
client.usdc.getAddress(optimism.id) // resolve by chain id
Decorators.getAddress(optimism.id, Addresses.usdc) // resolve from any address map
Addresses.usdc[mainnet.id] // raw map access (undefined if missing)
```

### Use on an Unsupported Chain

If the token's address cannot be resolved from the client (it's missing from the address map, or the
client has no chain), every action requires you to supply **either** an explicit `address` **or** a
`chain` to resolve the address from the token's own address map. This is enforced at the type level.

Pass a `chain` when the token has a configured address for that chain:

```ts twoslash
import { createClient, http } from 'viem'
import { optimism } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({ transport: http() }).extend(usdc()) // no chain

const hash = await client.usdc.transfer({
  chain: optimism, // resolve USDC from optimism.id
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  amount: '10.5',
})
```

Or pass an explicit `address` (and `decimals`, if it differs from the configured default, so string
`amount` inputs are parsed correctly):

```ts twoslash
import { createClient, http } from 'viem'
import { privateKeyToAccount } from 'viem/accounts'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({
  account: privateKeyToAccount('0x...'),
  chain: mainnet,
  transport: http(),
}).extend(usdc())

const hash = await client.usdc.transfer({
  address: '0x...', // token address on this chain
  decimals: 18, // override if it differs from the configured default
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  amount: '10.5',
})
```

### Define a Custom Token

Any ERC-20 token can be defined with [`Decorators.defineErc20`](#define-a-custom-token) by supplying a
namespace and its per-chain address map. The returned decorator has the exact same shape as `usdc`,
with no duplicated action code.

```ts twoslash
import { createClient, http } from 'viem'
import { mainnet } from 'viem/chains'
import { Decorators } from 'viem/tokens'

const dai = Decorators.defineErc20('dai', {
  addresses: { [mainnet.id]: '0x6B175474E89094C44Da98b954EedeAC495271d0F' },
  decimals: 18,
})

const client = createClient({
  chain: mainnet,
  transport: http(),
}).extend(dai())

const balance = await client.dai.getBalance({
  account: '0x55FE002aefF02F77364de339a1292923A15844B8',
})

// `amount` strings are parsed with the configured `decimals` (18 for DAI).
const hash = await client.dai.transfer({
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  amount: '10.5',
})
```

### Override Addresses

The decorator factory accepts optional `addresses` (merged over the configured map) and `decimals`
overrides. This works for built-in tokens like `usdc` too: supply an `address` for a chain that
isn't in the default map (or replace an existing one).

```ts twoslash
import { createClient, http } from 'viem'
import { mainnet } from 'viem/chains'
import { usdc } from 'viem/tokens'

const client = createClient({
  chain: mainnet,
  transport: http(),
}).extend(
  usdc({
    addresses: { [mainnet.id]: '0x...' }, // replace the default USDC address
    decimals: 6,
  }),
)
```

## Best Practices

- **`amount` is a human-readable string.** Pass a decimal string (e.g. `'10.5'`); it is parsed with
  the token's configured `decimals` into base units.
- **Set a `chain` on your client.** Token addresses are resolved from `client.chain.id`. If the
  client has no chain, or the token is not deployed on it, pass either an explicit `address` (and
  `decimals`, if it differs) or a `chain` to resolve the address from the token's address map.
- **Prefer `Sync` actions when you need the result.** `transferSync` and `approveSync` wait for
  confirmation and return the decoded event plus receipt.
- **Use `from` to transfer on behalf of another account.** Passing `from` to `transfer`/`transferSync`
  calls `transferFrom` using an allowance.
- **Use `.call` for composition.** Pass it to `estimateContractGas`, `simulateContract`,
  `sendCalls`, or `multicall` to estimate, simulate, or batch operations.
