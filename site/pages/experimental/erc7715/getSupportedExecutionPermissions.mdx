# getSupportedExecutionPermissions

Returns the supported execution permission types by the wallet.

[Read more.](https://eips.ethereum.org/EIPS/eip-7715)

:::warning[Warning]
This is an experimental action that is not supported in most wallets. It is recommended to have a fallback mechanism if using this in production.
:::

## Usage

:::code-group

```ts twoslash [example.ts]
import { walletClient } from './config'
 
const result = await walletClient.getSupportedExecutionPermissions() // [!code focus]
```

```ts twoslash [config.ts]
import { createWalletClient, custom } from 'viem'
import { mainnet } from 'viem/chains'
import { erc7715Actions } from 'viem/experimental'

export const walletClient = createWalletClient({
  chain: mainnet,
  transport: custom(window.ethereum!),
}).extend(erc7715Actions())
```

:::

## Returns

`GetSupportedExecutionPermissionsReturnType`

A record of supported permission types, where each key is a permission type and the value contains list of supported chain IDs, and rules.

### Example Response

```ts
{
  'native-token-allowance': {
    chainIds: [1, 137],
    ruleTypes: ['expiry']
  },
  'erc20-token-allowance': {
    chainIds: [1],
    ruleTypes: []
  }
}
```
