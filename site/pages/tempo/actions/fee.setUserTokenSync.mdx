# fee.setUserTokenSync

Sets the user's default fee token and waits for the transaction to be confirmed.

## Usage

:::code-group

```ts twoslash [Decorator]
import { client } from './viem.config'

const { token, user, receipt } = await client.fee.setUserTokenSync({
  token: 1n,
})
// @log: { token: '0x20C0000000000000000000000000000000000001', user: '0x…', receipt: { … } }
```

```ts twoslash [Standalone]
import { Actions } from 'viem/tempo'
import { client } from './viem.config'

const { token, user, receipt } = await Actions.fee.setUserTokenSync(client, {
  token: 1n,
})
// @log: { token: '0x20C0000000000000000000000000000000000001', user: '0x…', receipt: { … } }
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
import { Account } from 'viem'
import { Client } from 'viem/tempo'

export const client = Client.create({
  account: Account.fromPrivateKey('0x…'),
})
```

:::

## Parameters

### token

- **Type:** `TokenId | Address`

Address or ID of the TIP-20 token.

### throwOnReceiptRevert

- **Type:** `boolean`
- **Default:** `true`

Whether to throw when the receipt marks the transaction as reverted.

## Return Value

`{ token: Address, user: Address, receipt: TransactionReceipt }`

The `UserTokenSet` event data and the transaction receipt.

## Errors

| Error | Description |
| --- | --- |
| `ContractFunctionExecutionError` | The write failed or the receipt reverted. |
