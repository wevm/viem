# receivePolicy.claim

Claims blocked funds for a receipt, releasing them to a destination. Only the receipt's claimer (the original sender by default) can claim.

## Usage

:::code-group

```ts twoslash [Decorator]
import { client } from './viem.config'

const hash = await client.receivePolicy.claim({
  receipt: '0x…',
  to: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266',
})
// @log: '0x…'
```

```ts twoslash [Standalone]
import { Actions } from 'viem/tempo'
import { client } from './viem.config'

const hash = await Actions.receivePolicy.claim(client, {
  receipt: '0x…',
  to: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266',
})
// @log: '0x…'
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
import { Account } from 'viem'
import { Client } from 'viem/tempo'

export const client = Client.create({
  account: Account.fromPrivateKey('0x…'),
})
```

:::

## Parameters

### feeToken

- **Type:** `TokenId | Address`
- **Default:** `client.feeToken`

TIP-20 token to pay transaction fees with.

### nonceKey

- **Type:** `bigint | 'random'`

Nonce key to send the transaction under. Pass `'random'` to use a random key.

### receipt

- **Type:** `ReceivePolicyReceipt`

The encoded claim receipt (witness from a `TransferBlocked` event). The `ReceivePolicyReceipt` type is re-exported from `'viem/tempo'`.

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const hash = await Actions.receivePolicy.claim(client, {
  receipt: '0x…', // [!code focus]
  to: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266',
})
```

### to

- **Type:** `Address`

Destination to release the blocked funds to.

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const hash = await Actions.receivePolicy.claim(client, {
  receipt: '0x…',
  to: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266', // [!code focus]
})
```

## Return Value

`Hex`

The transaction hash.

## Errors

| Error | Description |
| --- | --- |
| `ContractFunctionExecutionError` | The write failed (for example, the receipt is invalid or the caller is not the authorized claimer). |
