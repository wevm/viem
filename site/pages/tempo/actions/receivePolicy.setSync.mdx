# receivePolicy.setSync

Sets the receive policy for the calling account, and waits for the transaction to be confirmed.

## Usage

:::code-group

```ts twoslash [Decorator]
import { client } from './viem.config'

const { claimer, senderPolicyId } = await client.receivePolicy.setSync({
  claimer: 'self',
  senderPolicyId: 'allow-all',
  tokenPolicyId: 'allow-all',
})
// @log: {
// @log:   account: '0x…',
// @log:   senderPolicyId: 'allow-all',
// @log:   tokenPolicyId: 'allow-all',
// @log:   claimer: 'self',
// @log:   recoveryAuthority: '0x…',
// @log:   receipt: { … },
// @log: }
```

```ts twoslash [Standalone]
import { Actions } from 'viem/tempo'
import { client } from './viem.config'

const { claimer, senderPolicyId } = await Actions.receivePolicy.setSync(
  client,
  {
    claimer: 'self',
    senderPolicyId: 'allow-all',
    tokenPolicyId: 'allow-all',
  },
)
// @log: {
// @log:   account: '0x…',
// @log:   senderPolicyId: 'allow-all',
// @log:   tokenPolicyId: 'allow-all',
// @log:   claimer: 'self',
// @log:   recoveryAuthority: '0x…',
// @log:   receipt: { … },
// @log: }
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
import { Account } from 'viem'
import { Client } from 'viem/tempo'

export const client = Client.create({
  account: Account.fromPrivateKey('0x…'),
})
```

## Parameters

### claimer

- **Type:** `'sender' | 'self' | Address`
- **Default:** `'sender'`

Who can reclaim funds blocked by this policy: the originator of the funds (`'sender'`), the account configuring the policy (`'self'`), or a delegated third party (an address).

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const result = await Actions.receivePolicy.setSync(client, {
  claimer: 'self', // [!code focus]
})
```

### feeToken

- **Type:** `TokenId | Address`
- **Default:** `client.feeToken`

TIP-20 token to pay transaction fees with.

### nonceKey

- **Type:** `bigint | 'random'`

Nonce key to send the transaction under. Pass `'random'` to use a random key.

### senderPolicyId

- **Type:** `'reject-all' | 'allow-all' | bigint`
- **Default:** `'allow-all'`

TIP-403 policy restricting which senders are allowed: the built-in reject-everything policy (`'reject-all'`, ID `0`), the built-in allow-everything policy (`'allow-all'`, ID `1`), or a custom policy ID (`>= 2`), such as one returned by [`policy.createSync`](/tempo/actions/policy.createSync).

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const result = await Actions.receivePolicy.setSync(client, {
  senderPolicyId: 2n, // [!code focus]
})
```

### throwOnReceiptRevert

- **Type:** `boolean`
- **Default:** `true`

Whether to throw when the receipt marks the transaction as reverted.

### tokenPolicyId

- **Type:** `'reject-all' | 'allow-all' | bigint`
- **Default:** `'allow-all'`

TIP-403 policy restricting which tokens are allowed.

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const result = await Actions.receivePolicy.setSync(client, {
  tokenPolicyId: 'reject-all', // [!code focus]
})
```

## Return Value

`{ account: Address, senderPolicyId: PolicyRef, tokenPolicyId: PolicyRef, claimer: Claimer, recoveryAuthority: Address, receipt: TransactionReceipt }`

The `ReceivePolicyUpdated` event data and the transaction receipt.

## Errors

| Error | Description |
| --- | --- |
| `Account.NotFoundError` | No `account` was provided and the client has no account. |
| `ContractFunctionExecutionError` | The write failed or the receipt reverted. |
