# dex.createPairSync

Creates a new trading pair on the DEX, and waits for the transaction to be confirmed.

## Usage

:::code-group

```ts twoslash [Decorator]
import { client } from './viem.config'

const { key, receipt } = await client.dex.createPairSync({
  base: '0x20c0000000000000000000000000000000000001',
})
// @log: {
// @log:   key: '0x…',
// @log:   base: '0x20c0000000000000000000000000000000000001',
// @log:   quote: '0x20c0000000000000000000000000000000000000',
// @log:   receipt: { … },
// @log: }
```

```ts twoslash [Standalone]
import { Actions } from 'viem/tempo'
import { client } from './viem.config'

const { key, receipt } = await Actions.dex.createPairSync(client, {
  base: '0x20c0000000000000000000000000000000000001',
})
// @log: {
// @log:   key: '0x…',
// @log:   base: '0x20c0000000000000000000000000000000000001',
// @log:   quote: '0x20c0000000000000000000000000000000000000',
// @log:   receipt: { … },
// @log: }
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
import { Account } from 'viem'
import { Client } from 'viem/tempo'

export const client = Client.create({
  account: Account.fromPrivateKey('0x…'),
})
```

:::

## Parameters

### base

- **Type:** `TokenId | Address`

Address or ID of the base token for the pair.

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const result = await Actions.dex.createPairSync(client, {
  base: 1n, // [!code focus]
})
```

### feeToken

- **Type:** `TokenId | Address`
- **Default:** `client.feeToken`

TIP-20 token to pay transaction fees with.

### nonceKey

- **Type:** `bigint | 'random'`

Nonce key to send the transaction under. Pass `'random'` to use a random key.

### throwOnReceiptRevert

- **Type:** `boolean`
- **Default:** `true`

Whether to throw when the receipt marks the transaction as reverted.

## Return Value

`{ key: Hex, base: Address, quote: Address, receipt: TransactionReceipt }`

The `PairCreated` event data (orderbook pair key and token addresses) and the transaction receipt.

## Errors

| Error | Description |
| --- | --- |
| `ContractFunctionExecutionError` | The write failed or the receipt reverted. |
