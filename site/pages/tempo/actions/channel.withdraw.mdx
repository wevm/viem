# `channel.withdraw`

Withdraws payer funds after the close grace period elapses.

## Usage

:::code-group

```ts twoslash [example.ts]
import { client, channel } from './viem.config'

const { receipt, refundedToPayer, settledToPayee } =
  await client.channel.withdrawSync({
    channel,
  })

console.log('Settled:', settledToPayee)
console.log('Refunded:', refundedToPayer)
console.log('Transaction hash:', receipt.transactionHash)
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
// [!include ~/snippets/tempo/viem.config.ts:setup]
// [!include ~/snippets/tempo/viem.config.ts:channel]
```

:::

Call `channel.requestClose` first, then wait for the close grace period before calling `channel.withdraw`.

### Asynchronous Usage

The example above uses a `*Sync` variant of the action, that will wait for the transaction to be included before returning.

If you are optimizing for performance, you should use the non-sync `channel.withdraw` action and wait for inclusion manually:

```ts twoslash
import { Actions } from 'viem/tempo'
import { client, channel } from './viem.config'

const hash = await client.channel.withdraw({
  channel,
})
const receipt = await client.waitForTransactionReceipt({ hash })

const { args } = Actions.channel.withdraw.extractEvent(receipt.logs)
```

## Return Type

```ts
type ReturnType = {
  /** Channel ID */
  channelId: Hex
  /** Account that receives settled voucher payments */
  payee: Address
  /** Account that funded the channel and receives refunds */
  payer: Address
  /** Amount refunded to the payer */
  refundedToPayer: bigint
  /** Transaction receipt */
  receipt: TransactionReceipt
  /** Amount settled to the payee */
  settledToPayee: bigint
}
```

## Parameters

### channel

- **Type:** `Channel.from.Value`

Channel containing the payer, payee, operator, token, salt, authorized signer, and expiring nonce hash.
