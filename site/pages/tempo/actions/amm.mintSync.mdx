# amm.mintSync

Adds liquidity to a fee AMM pool, and waits for the transaction to be confirmed. The fee AMM is single-sided: liquidity is provided in the validator token.

## Usage

:::code-group

```ts twoslash [Decorator]
import { client } from './viem.config'

const { liquidity, receipt } = await client.amm.mintSync({
  to: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266',
  userTokenAddress: '0x20c0000000000000000000000000000000000001',
  validatorTokenAddress: '0x20c0000000000000000000000000000000000002',
  validatorTokenAmount: 100n,
})
// @log: {
// @log:   sender: '0x…',
// @log:   to: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266',
// @log:   userToken: '0x20c0000000000000000000000000000000000001',
// @log:   validatorToken: '0x20c0000000000000000000000000000000000002',
// @log:   amountValidatorToken: 100n,
// @log:   liquidity: 100n,
// @log:   receipt: { … },
// @log: }
```

## Parameters

### feeToken

- **Type:** `TokenId | Address`
- **Default:** `client.feeToken`

TIP-20 token to pay transaction fees with.

### nonceKey

- **Type:** `bigint | 'random'`

Nonce key to send the transaction under. Pass `'random'` to use a random key.

### throwOnReceiptRevert

- **Type:** `boolean`
- **Default:** `true`

Whether to throw when the receipt marks the transaction as reverted.

### to

- **Type:** `Address`

Address to mint LP tokens to.

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const result = await Actions.amm.mintSync(client, {
  to: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266', // [!code focus]
  userTokenAddress: '0x20c0000000000000000000000000000000000001',
  validatorTokenAddress: '0x20c0000000000000000000000000000000000002',
  validatorTokenAmount: 100n,
})
```

### userTokenAddress

- **Type:** `TokenId | Address`

Address or ID of the user token of the pool.

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const result = await Actions.amm.mintSync(client, {
  to: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266',
  userTokenAddress: 1n, // [!code focus]
  validatorTokenAddress: '0x20c0000000000000000000000000000000000002',
  validatorTokenAmount: 100n,
})
```

### validatorTokenAddress

- **Type:** `TokenId | Address`

Address or ID of the validator token of the pool.

```ts twoslash

```

### validatorTokenAmount

- **Type:** `bigint`

Amount of validator token to add.

## Return Value

`{ sender: Address, to: Address, userToken: Address, validatorToken: Address, amountValidatorToken: bigint, liquidity: bigint, receipt: TransactionReceipt }`

The `Mint` event data and the transaction receipt.

## Errors

| Error | Description |
| --- | --- |
| `ContractFunctionExecutionError` | The write failed or the receipt reverted. |
