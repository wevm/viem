# `zone.waitForDepositStatus`

Waits for a Tempo block's deposits to be processed by a zone.

Polls [`zone.getDepositStatus`](/tempo/actions/zone.getDepositStatus) until the block has been processed.

## Usage

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const status = await client.zone.waitForDepositStatus({
  tempoBlockNumber: 42n,
})

console.log('Processed:', status.processed)
console.log('Deposits:', status.deposits)
// @log: Processed: true
// @log: Deposits: [{ amount: 100000000n, kind: 'regular', status: 'processed', ... }]
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
// [!include ~/snippets/tempo/viem.config.ts:setup]
```

:::

## Return Type

```ts
type ReturnType = {
  /** Deposits included in the block. */
  deposits: readonly {
    amount: bigint
    depositHash: Hex
    kind: 'encrypted' | 'regular'
    memo: Hex | null
    recipient: Address | null
    sender: Address
    status: 'failed' | 'pending' | 'processed'
    token: Address
  }[]
  /** Whether the block has been fully processed by the zone. */
  processed: boolean
  /** Queried parent chain block number. */
  tempoBlockNumber: bigint
  /** Latest parent chain block the zone has processed. */
  zoneProcessedThrough: bigint
}
```

The processed deposit status.

## Parameters

### pollingInterval (optional)

- **Type:** `number`
- **Default:** `client.pollingInterval`

Polling frequency (in ms).

### tempoBlockNumber

- **Type:** `bigint`

Parent Tempo chain block number to query.

### timeout (optional)

- **Type:** `number`
- **Default:** `60_000`

Timeout (in ms). Pass `0` to disable.

## Errors

| Error | Description |
| --- | --- |
| `WaitForDepositStatusTimeoutError` | Thrown when waiting for zone deposits times out. |
