# `earn.withdrawExact`

Withdraws an exact asset amount while limiting the EarnShare burned.

## Usage

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const result = await client.earn.withdrawExactSync({
  assetAmount: 40_000_000n,
  slippageBps: 50,
  earnVault: '0x0000000000000000000000000000000000000001',
})
// @log: {
//   assetAmount: 40_000_000n,
//   caller: '0x...',
//   receipt: { ... },
//   recipient: '0x...',
//   earnShares: 40_100_000n,
// }
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
// [!include ~/snippets/tempo/viem.config.ts:setup]
```

:::

When only `slippageBps` is provided, the action gets a live
[`earn.getWithdrawQuote`](/tempo/actions/earn.getWithdrawQuote) and derives the maximum EarnShare
input. The transaction approves and burns the required shares atomically.

### Set an Exact Maximum

Use `maxEarnShares` when you already know the most EarnShare you will burn.

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const result = await client.earn.withdrawExactSync({
  assetAmount: 40_000_000n,
  maxEarnShares: 40_250_000n,
  earnVault: '0x0000000000000000000000000000000000000001',
})
// @log: { assetAmount: 40_000_000n, earnShares: 40_100_000n, ... }
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
// [!include ~/snippets/tempo/viem.config.ts:setup]
```

:::

### Asynchronous Usage

`earn.withdrawExactSync` waits for inclusion and returns the decoded withdrawal event. Use
`earn.withdrawExact` to return the transaction hash immediately.

```ts twoslash
import { client } from './viem.config'

const hash = await client.earn.withdrawExact({
  assetAmount: 40_000_000n,
  maxEarnShares: 40_250_000n,
  earnVault: '0x0000000000000000000000000000000000000001',
})
// @log: 0x1234...abcd
```

## Return Value

```ts
type ReturnValue = {
  /** Exact assets received. */
  assetAmount: bigint
  /** Withdrawing caller. */
  caller: Address
  /** Transaction receipt. */
  receipt: TransactionReceipt
  /** Asset recipient. */
  recipient: Address
  /** EarnShare burned. */
  earnShares: bigint
}
```

The asynchronous action returns the transaction hash instead.

## Parameters

### assetAmount

- **Type:** `bigint | { decimals?: number | undefined; formatted: string }`

Exact assets to receive. A `bigint` uses base units; a formatted value uses the vault asset's
decimals unless `decimals` is provided.

### recipient (optional)

- **Type:** `Address`
- **Default:** `account.address`

Asset recipient.

### earnShares (optional)

- **Type:** `bigint`

Quoted EarnShare input. Provide it with `slippageBps` to avoid a live quote.

### maxEarnShares (optional)

- **Type:** `bigint`

Maximum EarnShare to burn. It cannot be combined with `earnShares` or `slippageBps`.

### slippageBps (optional)

- **Type:** `number`

Slippage headroom above the provided or live quote, in basis points. For example, `50` means 0.5%.

### earnVault

- **Type:** `Address`

EarnVault address.
