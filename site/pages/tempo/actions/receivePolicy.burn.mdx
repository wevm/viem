# receivePolicy.burn

Burns the funds backing a blocked receipt. Requires the caller to hold the token's `BURN_BLOCKED_ROLE`, and is only valid when the receipt's policy subject is currently unauthorized as a sender under the token's TIP-403 policy.

## Usage

:::code-group

```ts twoslash [Decorator]
import { client } from './viem.config'

const hash = await client.receivePolicy.burn({
  receipt: '0x…',
})
// @log: '0x…'
```

```ts twoslash [Standalone]
import { Actions } from 'viem/tempo'
import { client } from './viem.config'

const hash = await Actions.receivePolicy.burn(client, {
  receipt: '0x…',
})
// @log: '0x…'
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
import { Account } from 'viem'
import { Client } from 'viem/tempo'

export const client = Client.create({
  account: Account.fromPrivateKey('0x…'),
})
```

:::

## Parameters

### feeToken

- **Type:** `TokenId | Address`
- **Default:** `client.feeToken`

TIP-20 token to pay transaction fees with.

### nonceKey

- **Type:** `bigint | 'random'`

Nonce key to send the transaction under. Pass `'random'` to use a random key.

### receipt

- **Type:** `ReceivePolicyReceipt`

The encoded claim receipt (witness from a `TransferBlocked` event). The `ReceivePolicyReceipt` type is re-exported from `'viem/tempo'`.

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const hash = await Actions.receivePolicy.burn(client, {
  receipt: '0x…', // [!code focus]
})
```

## Return Value

`Hex`

The transaction hash.

## Errors

| Error | Description |
| --- | --- |
| `ContractFunctionExecutionError` | The write failed (for example, the receipt is invalid or the caller lacks `BURN_BLOCKED_ROLE`). |
