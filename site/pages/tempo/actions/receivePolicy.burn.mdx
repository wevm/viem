# `receivePolicy.burn`

:::warning
**Coming soon:** [TIP-1028](https://tips.sh/1028) ships in the **T6** hardfork — live on **Testnet** June 11 and **Mainnet** June 15.
:::

Burns the funds backing a blocked receipt. Requires the caller to hold the token's `BURN_BLOCKED_ROLE`, and is only valid when the receipt's policy subject is currently unauthorized as a sender under the token's TIP-403 policy. [Learn more about transfer policies](https://docs.tempo.xyz/protocol/tip403/spec)

## Usage

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const { receipt } = await client.receivePolicy.burnSync({
  receipt: '0x...',
})

console.log('Transaction hash:', receipt.transactionHash)
// @log: Transaction hash: 0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
// [!include ~/snippets/tempo/viem.config.ts:setup]
```

:::

### Asynchronous Usage

The example above uses the `burnSync` variant, which waits for the transaction to be included before returning.

If you are optimizing for performance, you should use the non-sync `receivePolicy.burn` action and wait for inclusion manually:

```ts twoslash
import { client } from './viem.config'

const hash = await client.receivePolicy.burn({
  receipt: '0x...',
})
const receipt = await client.waitForTransactionReceipt({ hash })
```

## Return Type

### burn

```ts
type ReturnType = Hash
```

Returns the transaction hash.

### burnSync

```ts
type ReturnType = {
  /** Event data from the `ReceiptBurned` event. */
  // ...
  /** Transaction receipt. */
  receipt: TransactionReceipt
}
```

Returns the `ReceiptBurned` event arguments along with the transaction receipt.

## Parameters

### receipt

- **Type:** `Hex`

The encoded claim receipt (the witness emitted by a [`TransferBlocked`](/tempo/actions/receivePolicy.watchBlocked) event).

### throwOnReceiptRevert (optional, `burnSync` only)

- **Type:** `boolean`
- **Default:** `true`

Whether to throw if the transaction's receipt status is reverted.
