# Overview

| Action | Description |
|--------|-------------|
| **AMM Actions** | |
| [`amm.burn`](/tempo/actions/amm.burn) | Removes liquidity from a fee AMM pool |
| [`amm.burnSync`](/tempo/actions/amm.burnSync) | Removes liquidity from a fee AMM pool, and waits for the transaction to be confirmed |
| [`amm.getLiquidityBalance`](/tempo/actions/amm.getLiquidityBalance) | Gets the LP token balance for an account in a specific pool |
| [`amm.getPool`](/tempo/actions/amm.getPool) | Gets the reserves for a liquidity pool |
| [`amm.mint`](/tempo/actions/amm.mint) | Adds liquidity to a fee AMM pool. The fee AMM is single-sided: liquidity is provided in the validator token |
| [`amm.mintSync`](/tempo/actions/amm.mintSync) | Adds liquidity to a fee AMM pool, and waits for the transaction to be confirmed. The fee AMM is single-sided: liquidity is provided in the validator token |
| [`amm.rebalanceSwap`](/tempo/actions/amm.rebalanceSwap) | Performs a rebalance swap from validator token to user token |
| [`amm.rebalanceSwapSync`](/tempo/actions/amm.rebalanceSwapSync) | Performs a rebalance swap from validator token to user token, and waits for the transaction to be confirmed |
| [`amm.watchBurn`](/tempo/actions/amm.watchBurn) | Watches for liquidity burn events, returning a watcher handle |
| [`amm.watchMint`](/tempo/actions/amm.watchMint) | Watches for liquidity mint events, returning a watcher handle |
| [`amm.watchRebalanceSwap`](/tempo/actions/amm.watchRebalanceSwap) | Watches for rebalance swap events, returning a watcher handle |
| **Channel Actions** | |
| [`channel.close`](/tempo/actions/channel.close) | Closes a TIP-20 payment channel from the payee or operator side |
| [`channel.closeSync`](/tempo/actions/channel.closeSync) | Closes a TIP-20 payment channel from the payee or operator side, and waits for the transaction to be confirmed |
| [`channel.getStates`](/tempo/actions/channel.getStates) | Gets TIP-20 payment channel state for a channel ID or channel descriptor |
| [`channel.open`](/tempo/actions/channel.open) | Opens and funds a TIP-20 payment channel |
| [`channel.openSync`](/tempo/actions/channel.openSync) | Opens and funds a TIP-20 payment channel, and waits for the transaction to be confirmed |
| [`channel.requestClose`](/tempo/actions/channel.requestClose) | Starts the payer close timer for a TIP-20 payment channel. Once the close grace period elapses, the payer can reclaim remaining funds with [`channel.withdraw`](/tempo/actions/channel.withdraw) |
| [`channel.requestCloseSync`](/tempo/actions/channel.requestCloseSync) | Starts the payer close timer for a TIP-20 payment channel, and waits for the transaction to be confirmed |
| [`channel.settle`](/tempo/actions/channel.settle) | Settles a TIP-20 payment channel voucher, paying the payee up to the voucher's cumulative amount |
| [`channel.settleSync`](/tempo/actions/channel.settleSync) | Settles a TIP-20 payment channel voucher, and waits for the transaction to be confirmed |
| [`channel.signVoucher`](/tempo/actions/channel.signVoucher) | Signs a TIP-20 payment channel voucher. Vouchers commit to a cumulative amount: each voucher covers the total paid over the channel's lifetime and supersedes earlier ones |
| [`channel.topUp`](/tempo/actions/channel.topUp) | Adds deposit to a TIP-20 payment channel |
| [`channel.topUpSync`](/tempo/actions/channel.topUpSync) | Adds deposit to a TIP-20 payment channel, and waits for the transaction to be confirmed |
| [`channel.withdraw`](/tempo/actions/channel.withdraw) | Withdraws payer funds after the close grace period elapses. Start the grace period with [`channel.requestClose`](/tempo/actions/channel.requestClose); the withdrawal is gated until it ends |
| [`channel.withdrawSync`](/tempo/actions/channel.withdrawSync) | Withdraws payer funds after the close grace period elapses, and waits for the transaction to be confirmed |

| [`policy.modifyBlacklistSync`](/tempo/actions/policy.modifyBlacklistSync) | Modifies a policy blacklist, and waits for the transaction to be confirmed |
| [`policy.modifyWhitelist`](/tempo/actions/policy.modifyWhitelist) | Modifies a policy whitelist, allowing or disallowing an account. Only the policy admin can modify a policy |

| [`policy.setAdminSync`](/tempo/actions/policy.setAdminSync) | Sets the admin for a policy, and waits for the transaction to be confirmed |
| [`policy.watchAdminUpdated`](/tempo/actions/policy.watchAdminUpdated) | Watches for policy admin update events, returning a watcher handle |
| [`policy.watchBlacklistUpdated`](/tempo/actions/policy.watchBlacklistUpdated) | Watches for blacklist update events, returning a watcher handle |

| **Receive Policy Actions** | |
| [`receivePolicy.burn`](/tempo/actions/receivePolicy.burn) | Burns the funds backing a blocked receipt. Requires the caller to hold the token's `BURN_BLOCKED_ROLE`, and is only valid when the receipt's policy subject is currently unauthorized as a sender under the token's TIP-403 policy |
| [`receivePolicy.burnSync`](/tempo/actions/receivePolicy.burnSync) | Burns the funds backing a blocked receipt, and waits for the transaction to be confirmed |
| [`receivePolicy.claim`](/tempo/actions/receivePolicy.claim) | Claims blocked funds for a receipt, releasing them to a destination. Only the receipt's claimer (the original sender by default) can claim |
| [`receivePolicy.claimSync`](/tempo/actions/receivePolicy.claimSync) | Claims blocked funds for a receipt, and waits for the transaction to be confirmed |
| [`receivePolicy.get`](/tempo/actions/receivePolicy.get) | Gets the receive policy configured for an account. A receive policy controls which TIP-20 tokens and which senders the account accepts |
| [`receivePolicy.getBlockedBalance`](/tempo/actions/receivePolicy.getBlockedBalance) | Gets the blocked balance for an encoded receipt: the amount held by the `ReceivePolicyGuard` that the receipt can claim or burn |
| [`receivePolicy.set`](/tempo/actions/receivePolicy.set) | Sets the receive policy for the calling account. A receive policy controls which TIP-20 tokens and which senders an account accepts. Inbound transfers and mints that violate the policy are not reverted; instead the funds are redirected to the `ReceivePolicyGuard` and can be reclaimed later with [`receivePolicy.claim`](/tempo/actions/receivePolicy.claim) |
| [`receivePolicy.setSync`](/tempo/actions/receivePolicy.setSync) | Sets the receive policy for the calling account, and waits for the transaction to be confirmed |
| [`receivePolicy.validate`](/tempo/actions/receivePolicy.validate) | Checks whether a transfer or mint to a receiver is allowed by the receiver's receive policy |
| [`receivePolicy.watchBlocked`](/tempo/actions/receivePolicy.watchBlocked) | Watches for blocked transfer events, returning a watcher handle. Each `TransferBlocked` log carries the encoded receipt used to claim or burn the blocked funds |
| [`receivePolicy.watchBurned`](/tempo/actions/receivePolicy.watchBurned) | Watches for receipt burned events, returning a watcher handle |
| [`receivePolicy.watchClaimed`](/tempo/actions/receivePolicy.watchClaimed) | Watches for receipt claimed events, returning a watcher handle |
| [`receivePolicy.watchUpdated`](/tempo/actions/receivePolicy.watchUpdated) | Watches for receive policy update events, returning a watcher handle |
