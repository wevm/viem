# receivePolicy.set

Sets the receive policy for the calling account. A receive policy controls which TIP-20 tokens and which senders an account accepts. Inbound transfers and mints that violate the policy are not reverted; instead the funds are redirected to the `ReceivePolicyGuard` and can be reclaimed later with [`receivePolicy.claim`](/tempo/actions/receivePolicy.claim).

## Usage

:::code-group

```ts twoslash [Decorator]
import { client } from './viem.config'

const hash = await client.receivePolicy.set({
  claimer: 'self',
  senderPolicyId: 'allow-all',
  tokenPolicyId: 'allow-all',
})
// @log: '0x…'
```

```ts twoslash [Standalone]
import { Actions } from 'viem/tempo'
import { client } from './viem.config'

const hash = await Actions.receivePolicy.set(client, {
  claimer: 'self',
  senderPolicyId: 'allow-all',
  tokenPolicyId: 'allow-all',
})
// @log: '0x…'
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
import { Account } from 'viem'
import { Client } from 'viem/tempo'

export const client = Client.create({
  account: Account.fromPrivateKey('0x…'),
})
```

:::

## Parameters

### claimer

- **Type:** `'sender' | 'self' | Address`
- **Default:** `'sender'`

Who can reclaim funds blocked by this policy: the originator of the funds (`'sender'`), the account configuring the policy (`'self'`), or a delegated third party (an address).

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const hash = await Actions.receivePolicy.set(client, {
  claimer: 'self', // [!code focus]
})
```

### feeToken

- **Type:** `TokenId | Address`
- **Default:** `client.feeToken`

TIP-20 token to pay transaction fees with.

### nonceKey

- **Type:** `bigint | 'random'`

Nonce key to send the transaction under. Pass `'random'` to use a random key.

### senderPolicyId

- **Type:** `'reject-all' | 'allow-all' | bigint`
- **Default:** `'allow-all'`

TIP-403 policy restricting which senders are allowed: the built-in reject-everything policy (`'reject-all'`, ID `0`), the built-in allow-everything policy (`'allow-all'`, ID `1`), or a custom policy ID (`>= 2`), such as one returned by [`policy.createSync`](/tempo/actions/policy.createSync).

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const hash = await Actions.receivePolicy.set(client, {
  senderPolicyId: 2n, // [!code focus]
})
```

### tokenPolicyId

- **Type:** `'reject-all' | 'allow-all' | bigint`
- **Default:** `'allow-all'`

TIP-403 policy restricting which tokens are allowed.

```ts twoslash
import { Actions } from 'viem/tempo'
import { client } from './viem.config'
// ---cut---
const hash = await Actions.receivePolicy.set(client, {
  tokenPolicyId: 'reject-all', // [!code focus]
})
```

## Return Value

`Hex`

The transaction hash.

| Error | Description |
