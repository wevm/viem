# Tempo

## Overview

The `viem/tempo` entrypoint is a self-contained kit for the [Tempo](https://tempo.xyz) blockchain: a Client factory with Tempo defaults, action namespaces for the Tempo precompiles (TIP-20 tokens, fee tokens, 2D nonces), and the support modules (`Abis`, `Addresses`, `Selectors`, `Scopes`, and more) the ecosystem builds on.

```ts twoslash
import { Client } from 'viem/tempo'

const client = Client.create()

const balance = await client.token.getBalance({
  account: '0x20c0000000000000000000000000000000000001',
  token: 1n,
})
```

`Client.create` defaults to the Tempo mainnet chain, an `http` transport, and the declared Tempo token set. Pass `testnet` to target the Tempo testnet, `chain` to override the chain entirely, or `feeToken` to set a default fee token for every transaction sent with the Client:

```ts twoslash
import { Account } from 'viem'
import { Client, http } from 'viem/tempo'

const client = Client.create({
  account: Account.fromPrivateKey('0x…'),
  feeToken: '0x20c0000000000000000000000000000000000001',
  testnet: true,
  transport: http('https://rpc.example.com'),
})
```

The returned Client is decorated with `publicActions`, `walletActions`, and `tempoActions`. Tempo namespaces merge with the core ones: `client.fee` carries both the core gas-fee actions and the Tempo fee-token actions.

:::tip
Tempo actions are also callable standalone via the `Actions` namespace: `Actions.token.transferSync(client, …)` with `import { Actions } from 'viem/tempo'`.
:::
