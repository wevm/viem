# Client [Setting up your Viem Client with Tempo Zones]

To use Tempo zone RPCs, extend your client with `tempoActions()`. On wallet-capable Tempo clients whose chain has configured `zones`, the client exposes `getZoneClient`.

## Usage

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const zoneClient = client.getZoneClient({
  zone: 26,
})

const info = await zoneClient.zone.getZoneInfo()
const balance = await zoneClient.token.getBalance({
  token: '0x20c0000000000000000000000000000000000000',
})

console.log('Zone ID:', info.zoneId)
// @log: Zone ID: 26
console.log('Zone chain ID:', info.chainId)
// @log: Zone chain ID: 4217000026
console.log('Zone balance:', balance)
// @log: Zone balance: 1000000n
```

## `getZoneClient`

Creates a wallet-capable client for a Tempo zone configured on the current chain.

By default, the returned client:

- Reuses the parent client's HTTP transport settings where possible.
- Disables HTTP batching because zone authorization tokens are account-scoped.
- Automatically signs and caches short-lived zone authorization tokens when using the built-in HTTP transport.
- Exposes Public Actions and Wallet Actions directly on `zoneClient`.
- Exposes Tempo token helpers on `zoneClient.token.*`.
- Exposes zone RPC helpers on `zoneClient.zone.*`.

Passing a transport factory via `transport` fully overrides the default zone HTTP transport.

### Parameters

#### zone

- **Type:** `number`

Zone ID configured on the parent chain.

#### transport (optional)

- **Type:** `Transport | ZoneTransportConfig`

Optional transport configuration for the derived zone client.

- Pass an object to override the built-in auth-aware HTTP transport, such as `timeout`, `retryCount`, `retryDelay`, `fetchOptions`, `key`, `methods`, or `name`.
- Pass a transport factory to fully override zone transport behavior. Explicit transport overrides are responsible for any zone-specific authentication they need.

## Return Type

```ts
type ReturnType = Client & {
  // Public Actions & Wallet Actions are available directly on the client.
  token: {
    approve: (parameters: {
      amount: bigint
      spender: Address
      token: Address
    }) => Promise<Hex>
    approveSync: (parameters: {
      amount: bigint
      spender: Address
      token: Address
    }) => Promise<unknown>
    getAllowance: (parameters: {
      spender: Address
      token: Address
    }) => Promise<bigint>
    getBalance: (parameters: {
      token: Address
    }) => Promise<bigint>
    getMetadata: (parameters: {
      token: Address
    }) => Promise<{
      currency?: string | undefined
      decimals: number
      name: string
      symbol: string
      totalSupply: bigint
    }>
    transfer: (parameters: {
      amount: bigint
      to: Address
      token: Address
    }) => Promise<Hex>
    transferSync: (parameters: {
      amount: bigint
      to: Address
      token: Address
    }) => Promise<unknown>
  }
  zone: {
    getAuthorizationTokenInfo: () => Promise<{
      account: Address
      expiresAt: bigint
    }>
    getDepositStatus: (parameters: {
      tempoBlockNumber: bigint
    }) => Promise<{
      deposits: readonly {
        amount: bigint
        depositHash: Hex
        kind: 'encrypted' | 'regular'
        memo: Hex | null
        recipient: Address | null
        sender: Address
        status: 'failed' | 'pending' | 'processed'
        token: Address
      }[]
      processed: boolean
      tempoBlockNumber: bigint
      zoneProcessedThrough: bigint
    }>
    getZoneInfo: () => Promise<{
      chainId: number
      sequencer: Address
      zoneId: number
      zoneTokens: readonly Address[]
    }>
  }
}
```

## Requirements

`getZoneClient` is only available when:

- The client is extended with `tempoActions()`.
- The client has wallet actions.
- The client has a signing account.
- The selected chain has configured `zones`.

## Related Exports

The same zone RPC helpers are also exported as free functions via `Actions.zone.*`
and `zoneActions.*`.

The following zone types are exported from `viem/tempo`:

- `GetZoneClientParameters`
- `ZoneChain`
- `ZoneClient`
- `ZoneId`
- `ZoneTransportConfig`
