# Errors

## Overview

The **Errors** module provides the base error class inherited by every error thrown by Viem,
domain-grouped Action errors, and configuration for how those errors are attributed.

`Errors.BaseError` extends ox's `BaseError`, so every Viem error is also an ox error (`instanceof`
holds across the stack) and ox errors compose as `cause` without wrapping.

Action errors live under semantic modules such as `Errors.block.NotFoundError` and
`Errors.transaction.NotFoundError`, keeping the `Actions` namespace focused on executable APIs.

```ts twoslash
import { Errors } from 'viem'

throw new Errors.BaseError('An error occurred.', {
  docsPath: '/docs/errors',
  metaMessages: ['Reason: example'],
})
```
