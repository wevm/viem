# erc20.transferSync

Transfers ERC-20 tokens to a recipient, **waits for the transaction to be confirmed**, and returns the
decoded `Transfer` event along with the transaction receipt.

Pass `from` to transfer on behalf of another address using an allowance (calls `transferFrom`);
otherwise transfers from the caller (calls `transfer`).

## Usage

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const { from, to, value, formatted, receipt } = await client.erc20.transferSync({
  amount: '10.5', // 10.5 USDC
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  token: 'usdc',
})
```


:::

## Return Type

```ts
type ReturnType = {
  /** Token decimals used to derive `formatted`. */
  decimals: number
  /** Transferred amount, formatted with the token's `decimals`. */
  formatted: string
  /** Address tokens were transferred from. */
  from: Address
  /** Transaction receipt. */
  receipt: TransactionReceipt
  /** Address tokens were transferred to. */
  to: Address
  /** Amount of tokens transferred, in base units. */
  value: bigint
}
```

## Parameters

### amount

- **Type:** `string`

Amount of tokens to transfer, as a human-readable decimal string (e.g. `'10.5'`). Parsed with the
token's `decimals`.

### to

- **Type:** `Address`

Address to transfer tokens to.

### from (optional)

- **Type:** `Address`

Address to transfer tokens from, using an allowance (calls `transferFrom`). Defaults to the caller.

### token

- **Type:** `string`

Token to operate on: either the name of a token declared on the connected
chain's [`tokens` config](/docs/erc20#declaring-tokens-on-a-chain) (resolved to
its contract `address` and `decimals`), or a contract `address`.

### decimals (optional)

- **Type:** `number`

Token decimals, used to convert between base units and the human-readable
amount (parsing `amount` for writes, formatting `amount`/`formatted` for
reads). Inferred from the chain's `tokens` config when `token` matches a declared
token (by name or address). Defaults to `0` when the token isn't declared on the
connected chain and `decimals` is omitted.

### throwOnReceiptRevert (optional)

- **Type:** `boolean`
- **Default:** `true`

Whether to throw if the transaction receipt reverts.

### account (optional)

- **Type:** `Account | Address`

Account that will send the transaction. Defaults to the Client's `account`.

### gas (optional)

- **Type:** `bigint`

Gas limit for the transaction.

### maxFeePerGas (optional)

- **Type:** `bigint`

Max fee per gas for the transaction.

### maxPriorityFeePerGas (optional)

- **Type:** `bigint`

Max priority fee per gas for the transaction.

### nonce (optional)

- **Type:** `number`

Nonce for the transaction.
