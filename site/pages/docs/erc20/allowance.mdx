# erc20.allowance

Reads the amount of tokens a spender is allowed to spend on behalf of an owner.

## Usage

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const allowance = await client.erc20.allowance({
  owner: '0x55FE002aefF02F77364de339a1292923A15844B8',
  spender: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  token: 'usdc',
})
// @log: { amount: 1000000000n, decimals: 6, formatted: '1000' }
```

[`multicall`](/docs/contract/multicall), [`simulateContract`](/docs/contract/simulateContract), or any
other Action that accepts a contract call.

## Return Type

```ts
type ReturnType = {
  /** Allowance in the token's base units. */
  amount: bigint
  /** Token decimals used to derive `formatted`. */
  decimals: number
  /** Allowance formatted with the token's `decimals`. */
  formatted: string
}
```

The remaining allowance, both in base units (`amount`) and as a human-readable decimal string
(`formatted`, derived from the token's `decimals`).

## Parameters

### owner

- **Type:** `Address`

Account that owns the tokens.

### spender

- **Type:** `Address`

Account allowed to spend the owner's tokens.

### token

- **Type:** `string`

Token to operate on: either the name of a token declared on the connected
chain's [`tokens` config](/docs/erc20#declaring-tokens-on-a-chain) (resolved to
its contract `address` and `decimals`), or a contract `address`.

### decimals (optional)

- **Type:** `number`

Token decimals, used to convert between base units and the human-readable
amount (parsing `amount` for writes, formatting `amount`/`formatted` for
reads). Inferred from the chain's `tokens` config when `token` matches a declared
token (by name or address). Defaults to `0` when the token isn't declared on the
connected chain and `decimals` is omitted.
