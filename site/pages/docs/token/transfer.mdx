# token.transfer

Transfers ERC-20 tokens to a recipient, and returns the transaction hash.

Pass `from` to transfer on behalf of another address using an allowance (calls `transferFrom`);
otherwise transfers from the caller (calls `transfer`).

:::tip
Use [`transferSync`](/docs/token/transferSync) to wait for the transaction to be confirmed and return
the decoded `Transfer` event.
:::

## Usage

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const hash = await client.token.transfer({
  amount: '10.5', // 10.5 USDC
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  token: 'usdc',
})
```

### Transfer From an Approved Account

Pass `from` to move tokens on behalf of an owner that has approved the caller (calls `transferFrom`).

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const hash = await client.token.transfer({
  amount: '10.5',
  from: '0x55FE002aefF02F77364de339a1292923A15844B8',
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  token: 'usdc',
})
```

### Estimate Gas & Simulate

Every write Action exposes `.estimateGas` and `.simulate`, which take the same parameters as the
Action itself.

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const gas = await client.token.transfer.estimateGas({
  amount: '1',
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  token: 'usdc',
})

const { result } = await client.token.transfer.simulate({
  amount: '1',
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  token: 'usdc',
})
```

### Composing Calls

Use `.call` to build a `transfer` (or `transferFrom`, when `from` is given) contract call, ready to
pass to [`sendCalls`](/docs/actions/wallet/sendCalls), `sendTransaction` (`calls`), or
[`multicall`](/docs/contract/multicall).

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const call = client.token.transfer.call({
  amount: '1',
  to: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEbb',
  token: 'usdc',
})
```

## Return Type

[`Hex`](/docs/glossary/types#hex)

The transaction hash.

## Parameters

### amount

- **Type:** `string`

Amount of tokens to transfer, as a human-readable decimal string (e.g. `'10.5'`). Parsed with the
token's `decimals`.

### to

- **Type:** `Address`

Address to transfer tokens to.

### token

- **Type:** `string`

Token to operate on: a [chain-declared](/docs/token#declaring-tokens-on-a-chain)
token name (resolving its `address` and `decimals`), or a contract `address`.

### from (optional)

- **Type:** `Address`

Address to transfer tokens from, using an allowance (calls `transferFrom`). Defaults to the caller.

### decimals (optional)

- **Type:** `number`

Token decimals for converting base units to human-readable amounts. Inferred
from the chain's `tokens` config, defaulting to `0`.

### account (optional)

- **Type:** `Account | Address`

Account that will send the transaction. Defaults to the Client's `account`.

### gas (optional)

- **Type:** `bigint`

Gas limit for the transaction.

### maxFeePerGas (optional)

- **Type:** `bigint`

Max fee per gas for the transaction.

### maxPriorityFeePerGas (optional)

- **Type:** `bigint`

Max priority fee per gas for the transaction.

### nonce (optional)

- **Type:** `number`

Nonce for the transaction.
