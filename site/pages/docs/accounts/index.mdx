# Account

## Overview

The **Account** module represents an Ethereum account that can sign messages, transactions, typed data, and authorizations.

Viem distinguishes between two kinds of account:

- **Local Account**: signing happens locally from injected signing logic (a private key, mnemonic, KMS, etc.). The raw signing material is available to your program.
- **JSON-RPC Account**: signing is delegated to a JSON-RPC provider (for example, a browser wallet). Only the address is known.

## Reference

The `Account` module creates and represents accounts that sign messages, transactions, typed data, and authorizations.

### Usage

```ts twoslash
import { Account } from 'viem'

// A local account from a private key.
const local = Account.fromPrivateKey('0x...')

// A JSON-RPC account from an address (e.g. a browser wallet).
const jsonRpc = Account.from('0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266')
```

### Types

- `Account.Account`: a local signer, or a JSON-RPC account referenced by address.
- `Account.Local`: an account that signs locally from injected signing logic.
- `Account.JsonRpc`: an account whose signing is delegated to a JSON-RPC provider.
- `Account.PrivateKey`: a private-key-backed local account.
- `Account.Hd`: a mnemonic/HD-derived local account.
- `Account.CustomSource`: the custom signing logic backing a local account.
- `Account.SignableMessage`: a UTF-8 string, or a raw payload signed as-is.
