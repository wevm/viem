# Custom Accounts

## Overview

A **Custom Account** is a [local account](/docs/accounts) (`Account.Local<'custom'>`) backed by signing logic you provide, for example a KMS, an HSM, or a remote signing service.

Create one with [`Account.from`](/docs/accounts) by passing an account object. The account only requires an `address` (or a `publicKey` to derive it from) and a `sign` primitive; `signMessage`, `signTransaction`, `signTypedData`, and `signAuthorization` are derived from `sign` (and can be overridden when a backend needs custom payload handling).

```ts twoslash
import { Account } from 'viem'

// An external signing service (e.g. a KMS) that signs a hash and returns a hex signature.
declare const kms: { sign(hash: string): Promise<`0x${string}`> }

const account = Account.from({
  address: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266',
  async sign({ hash }) {
    return await kms.sign(hash)
  },
})
```

## `Account.from`

Creates an account from an address (JSON-RPC) or custom signing logic (local).

### Usage

```ts twoslash
import { Account } from 'viem'

// An external signing service (e.g. a KMS) that signs a hash and returns a hex signature.
declare const kms: { sign(hash: string): Promise<`0x${string}`> }

const account = Account.from({
  address: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266',
  async sign({ hash }) {
    return await kms.sign(hash)
  },
})
```

### Parameters

#### account

- **Type:** `Address.Address | Account.from.Account`

A JSON-RPC account address, or an account object backing a local account. The account requires a `sign` method plus either an `address` or a `publicKey` (the `address` is derived from `publicKey` when omitted); `signMessage` / `signTransaction` / `signTypedData` / `signAuthorization` are derived from `sign` when omitted, and can be overridden. `keyType` may also be supplied (defaulting to `'custom'`).

```ts twoslash
import { Account } from 'viem'
// ---cut---
// An external signing service (e.g. a KMS) that signs a hash and returns a hex signature.
declare const kms: { sign(hash: string): Promise<`0x${string}`> }

const account = Account.from({ // [!code focus]
  address: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266', // [!code focus]
  async sign({ hash }) { // [!code focus]
    return await kms.sign(hash) // [!code focus]
  }, // [!code focus]
}) // [!code focus]
```

### Return Value

`Account.Local<'custom'>`

A local account stamped with `keyType: 'custom'`.

### Errors

| Error                         | Description                                           |
| ----------------------------- | ----------------------------------------------------- |
| `Address.InvalidAddressError` | The supplied address is not a valid Ethereum address. |
| `Errors.GlobalErrorType`      | A globally-handled error.                             |
