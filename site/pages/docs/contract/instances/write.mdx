# contract.write

State-mutating (`nonpayable`/`payable`) contract methods, one per ABI function. Each maps to [`Actions.contract.write`](/docs/actions/wallet/contract/write) with `abi`, `address`, and `functionName` bound, and returns the transaction hash.

The `write` group requires an [Account](/docs/accounts): either on the Client, or passed per call via the `account` option (required on account-less Client types). Calls reject with `Account.NotFoundError` when neither is provided. [`simulate`](/docs/contract/instances/simulate) and [`estimateGas`](/docs/contract/instances/estimateGas) do not require one.

## Usage

```ts twoslash
import { Account, Client, Contract, http } from 'viem'
import { mainnet } from 'viem/chains'
import { Abis } from 'viem/utils'

const client = Client.create({
  account: Account.fromPrivateKey('0x...'),
  chain: mainnet,
  transport: http(),
})

const contract = Contract.from({
  abi: Abis.erc20,
  address: '0xFBA3912Ca04dd458c843e2EE08967fC04f3579c2',
  client,
})

const hash = await contract.write.transfer([ // [!code focus]
  '0x70997970c51812dc3a010c7d01b50e0d17dc79c8', // [!code focus]
  1n, // [!code focus]
]) // [!code focus]
// @log: '0x...'
```

## Options

Function arguments are passed as the first parameter. The second parameter accepts the options of [`Actions.contract.write`](/docs/actions/wallet/contract/write#parameters) minus the bound `abi`, `address`, `args`, and `functionName`, such as `value` (payable functions only) and transaction-request fields like `nonce` and fees.

When a function has no inputs, omit the arguments array and pass options as the first parameter.

## Return Value

`Promise<Hex>`

The transaction hash.
