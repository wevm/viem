# Defining a Chain

## Overview

Use [`Chain.from`](/docs/chains) to define a [Chain](/docs/chains). It preserves the literal type of the input (so ids, names, and contract addresses stay narrow) and attaches a chainable [`.extend()`](/docs/chains/extend) for deriving a new chain from a base.

### Required Fields

A chain requires the following fields; all others are optional.

| Field | Type | Description |
| --- | --- | --- |
| `id` | `number` | Chain id. |
| `name` | `string` | Human-readable name. |
| `nativeCurrency` | `Chain.NativeCurrency` | Currency used by the chain (`name`, `symbol`, `decimals`). |
| `rpcUrls` | `{ default: Chain.RpcUrls }` | RPC endpoints, keyed by name with a required `default`. |

### Optional Behavior

Beyond plain metadata, a chain carries two optional behavior slots:

- **`schema`**: bidirectional RPC to native codecs (the declarative replacement for function-based formatters).
- **`serializers`**: transaction serialization overrides.

## `Chain.from`

Defines a chain, preserving its literal type and attaching a chainable `.extend()`.

### Usage

```ts twoslash
import { Chain } from 'viem'

const mainnet = Chain.from({
  id: 1,
  name: 'Ethereum',
  nativeCurrency: { name: 'Ether', symbol: 'ETH', decimals: 18 },
  rpcUrls: { default: { http: ['https://eth.merkle.io'] } },
})
```

### Parameters

#### chain

- **Type:** `Chain.Chain`

The chain definition.

### Return Value

`chain & { extend: (overrides) => Chain }`

The chain with its literal type preserved, plus a chainable [`.extend()`](/docs/chains/extend).
