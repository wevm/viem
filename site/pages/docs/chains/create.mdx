# Defining a Chain

## Overview

Use [`Chain.from`](/docs/chains) to define a [Chain](/docs/chains). It preserves the literal type of the input (so ids, names, and contract addresses stay narrow) and attaches a chainable [`.extend()`](/docs/chains/extend) for deriving a new chain from a base.

## `Chain.from`

Defines a chain, preserving its literal type and attaching a chainable `.extend()`.

### Usage

```ts twoslash
import { Chain } from 'viem'

const mainnet = Chain.from({
  id: 1,
  name: 'Ethereum',
  nativeCurrency: { name: 'Ether', symbol: 'ETH', decimals: 18 },
  rpcUrls: { default: { http: ['https://eth.merkle.io'] } },
})
```

### Parameters

#### chain

- **Type:** `Chain.Chain`

The chain definition. `id`, `name`, `nativeCurrency`, and `rpcUrls` are required; all other fields are optional.

| Field | Type | Description |
| --- | --- | --- |
| `id` | `number` | Chain id. |
| `name` | `string` | Human-readable name. |
| `nativeCurrency` | `Chain.NativeCurrency` | Currency used by the chain (`name`, `symbol`, `decimals`). |
| `rpcUrls` | `{ default: Chain.RpcUrls }` | RPC endpoints, keyed by name with a required `default`. |
| `blockExplorers` | `{ default: Chain.BlockExplorer }` | Block explorers, keyed by name with a required `default`. |
| `blockTime` | `number` | Block time in milliseconds. |
| `contracts` | `Chain.Contracts` | Deployed contracts on the chain. |
| `ensTlds` | `readonly string[]` | ENS TLDs for the chain. |
| `preconfirmationTime` | `number` | Preconfirmation time in milliseconds. |
| `schema` | `Chain.Schema` | Bidirectional RPC to native codecs. |
| `serializers` | `Chain.Serializers` | Transaction serialization overrides. |
| `sourceId` | `number` | Source chain id (e.g. the L1 chain). |
| `testnet` | `boolean` | Flag for test networks. |

### Return Value

`chain & { extend: (overrides) => Chain }`

The chain with its literal type preserved, plus a chainable [`.extend()`](/docs/chains/extend).
