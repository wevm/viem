# Test Actions

## Overview

**Test Actions** manipulate a local test node such as [Anvil](https://book.getfoundry.sh/anvil/), [Hardhat](https://hardhat.org/), or [Ganache](https://github.com/trufflesuite/ganache). They let you mine blocks, set account balances and code, snapshot and revert state, impersonate accounts, and more, which is invaluable when writing tests.

Attach [`testActions`](#testactions) to a Client to call them as methods, or call each action standalone by passing a Client as the first argument from the `Actions` namespace:

:::code-group

```ts twoslash [Decorator]
import { client } from './viem.config'

await client.block.mine({ blocks: 1 })
```

```ts twoslash [Standalone]
import { Actions } from 'viem'
import { client } from './viem.config'

await Actions.block.mine(client, { blocks: 1 })
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
import { Client, http, testActions } from 'viem'
import { anvil } from 'viem/chains'

export const client = Client.create({
  chain: anvil,
  transport: http(),
}).extend(testActions())
```

:::

## Test Node Modes

Test methods are mode-prefixed (for example `anvil_setBalance`, `hardhat_setBalance`), so each action accepts a `mode` option (and [`testActions`](#testactions) accepts it for every action at once). Supported modes are `'anvil'` (default), `'hardhat'`, and `'ganache'`.

```ts twoslash
import { Client, http, testActions } from 'viem'
import { mainnet } from 'viem/chains'

const client = Client.create({
  chain: mainnet,
  transport: http(),
}).extend(
  testActions({ mode: 'hardhat' }), // [!code focus]
)
```

## testActions

Returns a bag of test actions bound to a Client, with the configured `mode` injected into every action. Pass it to [`.extend()`](/docs/clients/create#extending-a-client).

### Usage

```ts twoslash
import { Client, http, testActions } from 'viem'
import { mainnet } from 'viem/chains'

const client = Client.create({
  chain: mainnet,
  transport: http(),
}).extend(testActions())

await client.block.mine({ blocks: 1 })
```

### Parameters

#### mode

- **Type:** `'anvil' | 'hardhat' | 'ganache'`
- **Default:** `'anvil'`

The test node implementation to target. It is injected as the `mode` of every action on the decorator.

```ts twoslash
import { Client, http, testActions } from 'viem'
import { mainnet } from 'viem/chains'

const client = Client.create({
  chain: mainnet,
  transport: http(),
}).extend(
  testActions({ mode: 'hardhat' }), // [!code focus]
)
```

### Return Value

`(client: Client) => Decorator`

A function that decorates a Client with every test action as a method.
