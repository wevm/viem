# Defining Tokens

Token Actions can select tokens by token address, or by a symbol that is declared on the active
chain. Use `defineToken` when you want a reusable token definition, then attach that token to a
chain's `tokens` map to make symbol selectors like `token: 'usdc'` available.

## Define a Token

Use [`defineToken`](/tokens) to create a token from shared metadata (`decimals`, `name`, `symbol`) and
a per-chain `addresses` map. This step does not attach the token to a chain; it only creates a
reusable definition.

```ts twoslash
import { defineToken } from 'viem/tokens'

export const usdc = defineToken({ // [!code focus:9]
  addresses: {
    1: '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48',
    10: '0x0b2C639c533813f4Aa9D7837CAf62653d097Ff85',
  },
  decimals: 6,
  name: 'USD Coin',
  symbol: 'USDC',
})
```

The returned token is callable with a chain id when you need a chain token config.

```ts twoslash
import { mainnet } from 'viem/chains'
import { defineToken } from 'viem/tokens'

const usdc = defineToken({
  addresses: {
    1: '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48',
    10: '0x0b2C639c533813f4Aa9D7837CAf62653d097Ff85',
  },
  decimals: 6,
  name: 'USD Coin',
  symbol: 'USDC',
})
// ---cut---
const mainnetUsdc = usdc(mainnet.id)
// @log: {
// @log:   address: '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48',
// @log:   decimals: 6,
// @log:   name: 'USD Coin',
// @log:   symbol: 'USDC',
// @log: }
```

## Attach Tokens to a Chain

Attach token configs under the chain's [`tokens` property](/docs/chains/tokens). The object key is the
selector used by Token Actions.

```ts twoslash
import { defineChain } from 'viem'
import { defineToken } from 'viem/tokens'

const usdc = defineToken({
  addresses: {
    1: '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48',
  },
  decimals: 6,
  name: 'USD Coin',
  symbol: 'USDC',
})

export const example = defineChain({ // [!code focus:9]
  id: 1,
  name: 'Ethereum',
  nativeCurrency: { name: 'Ether', symbol: 'ETH', decimals: 18 },
  rpcUrls: { default: { http: ['https://eth.merkle.io'] } },
  tokens: {
    usdc: usdc(1), // [!code hl]
  },
})
```

You can also attach a one-off token config directly:

```ts twoslash
import { defineChain } from 'viem'

export const example = defineChain({
  id: 1,
  name: 'Ethereum',
  nativeCurrency: { name: 'Ether', symbol: 'ETH', decimals: 18 },
  rpcUrls: { default: { http: ['https://eth.merkle.io'] } },
  tokens: {
    usdc: { // [!code focus:6]
      address: '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48',
      decimals: 6,
      name: 'USD Coin',
      symbol: 'USDC',
    },
  },
})
```

Once attached, the token can be selected by the `tokens` map key on any Token Action.

```ts twoslash
import { createPublicClient, defineChain, http } from 'viem'

const example = defineChain({
  id: 1,
  name: 'Ethereum',
  nativeCurrency: { name: 'Ether', symbol: 'ETH', decimals: 18 },
  rpcUrls: { default: { http: ['https://eth.merkle.io'] } },
  tokens: {
    usdc: {
      address: '0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48',
      decimals: 6,
      name: 'USD Coin',
      symbol: 'USDC',
    },
  },
})

const client = createPublicClient({ chain: example, transport: http() })
// ---cut---
const balance = await client.token.getBalance({
  account: '0x55FE002aefF02F77364de339a1292923A15844B8',
  token: 'usdc', // [!code focus]
})
// @log: { amount: 1000000n, decimals: 6, formatted: '1' }
```
